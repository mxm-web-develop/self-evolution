# Maturity Assessment Prompt Template

## 目的
用于评估一个项目当前所处的成熟度阶段，识别其是否具备清晰目标、结构化资产、可执行路径与可持续演进能力。

## 输入变量
- `project_profile`: 项目画像，包含目标、背景、项目类型、约束、关键上下文。
- `investigation_result`: 调研结果，包含代码/结构扫描、外部发现、历史分析摘要等。
- `historical_memory`: 历史记忆，包含跨项目通用洞察、同类项目经验、近期会话沉淀。

## 推理要求
1. 先识别当前项目已具备的成熟信号。
2. 再识别缺失的关键能力、资料或机制。
3. 判断当前更接近哪个成熟阶段（例如：early / growing / mature），但不要依赖固定行业知识。
4. 说明评分依据，强调“为什么现在是这个阶段，而不是更高或更低”。
5. 输出必须可供程序读取，字段稳定。

## 输出格式
请严格输出 JSON：

```json
{
  "stage": "early|growing|mature",
  "label": "面向人类的中文标签",
  "score": 0,
  "reasoning": ["依据1", "依据2"],
  "strengths": ["当前已有的成熟信号"],
  "missing_capabilities": ["当前仍缺失的关键能力"],
  "success_signals": ["下一阶段应具备的信号"]
}
```

## 约束
- 不要写具体行业方案。
- 不要假设任何网站、量化、写作、AI 应用等特定业务。
- 只提供通用成熟度判断框架与结构化输出。
